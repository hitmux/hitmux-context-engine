# Documentation

Hitmux Context Engine documentation is intentionally flat. Start with configuration when you are setting up the product, then jump to the page that matches the task in front of you.

| Task | Page |
| --- | --- |
| Configure providers, Milvus/Zilliz, indexing, sync, and file filtering | [Configuration](configuration.md) |
| Add the MCP server to Claude Code, Codex CLI, OpenCode, VS Code MCP, or JSON-based clients | [Quick Start](quick-start.md) |
| Diagnose setup, proxy, reconnect, Windows command shim, or indexing status problems | [Troubleshooting](troubleshooting.md) |
| Inspect MCP tools, package aliases, and core TypeScript APIs | [Package Reference](package-reference.md) |
